# 矿山无人车场景下基于CenterPoint的3D点云目标检测优化与落地
## 项目职责
主导CenterPoint 3D检测框架在矿山恶劣场景的适配优化与工程落地，全流程负责训练策略迭代、数据增强方案设计、多模块融合开发，核心解决矿山场景下目标漏检、土墙/扬尘误检导致的无人车异常刹车问题，保障感知系统稳定适配下游规划控制模块。

## 核心工作与技术方案
1.  **场景化前置适配与去地图化改造**
    - 前期基于高精地图栅格mask，将0/1二值化可行驶区域特征与CenterPoint骨干网络输入特征拼接，过滤不可行驶区域误检；
    - 针对地图格式转换耗时、车端部署依赖重的问题，完成去地图化改造，通过数据与训练策略优化替代地图约束，降低部署链路复杂度。

2.  **GT-Sampler样本增强与类别均衡优化**
    - 深入CenterPoint数据链路，修改`config`中`db_sampler`模块与`sampler.py`采样逻辑，基于标注框8角点提取点云xyz+强度信息生成GT-DATABASE的.bin样本文件；
    - 针对矿山场景洒水车、加油车、钻孔机、充电站等罕见目标，扩充样本量从数十个至数百个，通过类别样本均衡策略避免单类别样本失衡，解决罕见目标漏检问题。

3.  **融合Pointcept地面分割解决误检问题**
    - 完成Pointcept（PPT）地面分割模型与CenterPoint训练链路的工程化融合，在`det3d.datasets.pipelines.preprocess.py`的Preprocess模块中新增地面点读取与对齐逻辑；
    - 基于分割模型输出的地面点掩码，通过KDTree匹配GT框最低点xy坐标对应的地面点z值，修正GT框与样本点云的高度坐标，确保扩增样本仅生成在可行驶地面上，从根源解决土墙堆误检车辆的问题。

4.  **GT-DATABASE精细化过滤与鲁棒性优化**
    - 针对矿山粉尘、远距离点云稀疏的特性，按类别设置精细化过滤阈值：行人≥25个点、小轿车≥150个点、矿卡运载车≥250个点，同时限制样本最远距离，过滤低质量、噪声标注样本；
    - 优化CenterPoint体素内点云采样策略，采用近到远优先级采样，适配矿山场景远距离点云稀疏、近距离点云稠密的特性，提升特征提取有效性。

## 量化成果
- 检测精度：核心类别矿卡AP从90%提升至93%，行人、挖机、铲车等罕见目标AP从60%提升至80%；
- 误检优化：土墙/扬尘导致的异常刹车频率下降80%以上，误检问题单从每1趟运输1次降至每5趟运输1次；
- 工程落地：完成全链路训练-推理流程的车端适配，交付的感知模块满足矿山无人车24小时连续作业要求，无感知误检导致的业务中断。
